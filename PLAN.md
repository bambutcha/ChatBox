# План выполнения тестового задания

## Анализ задачи

### Понятные части:
- **Дизайн**: Нужно воссоздать интерфейс Telegram чата с зеленым градиентным фоном, сообщениями справа, временем и чекмарками
- **Фронтенд**: React + TypeScript, стилизация через Tailwind/CSS
- **Бекенд**: Go 1.25.3, REST API, хранение сообщений в памяти
- **Функционал**: Отправка сообщений, сохранение на бекенде, отображение истории после перезагрузки
- **Деплой**: Публичный доступ к фронтенду и бекенду

### Требует уточнения/поиска:
- Точные оттенки зеленого градиента (нужно будет подобрать по скриншоту)
- Структура API (спроектирую самостоятельно, но нужно продумать формат)
- Платформа для деплоя (выберу подходящую)

### Что не нужно делать:
- Иконку эмодзи
- Иконку скрепки
- Иконку микрофона
- Дату сообщений (только время)
- Блок пустого чата

## Порядок выполнения

1. **Структура проекта и план** ✅
   - Создать структуру директорий (backend/, frontend/)
   - Написать план работы

2. **Бекенд (Go)**
   - Инициализировать Go модуль
   - Создать простой REST API сервер
   - Реализовать хранение сообщений в памяти (map или slice)
   - API endpoints:
     - `GET /api/messages` - получить все сообщения
     - `POST /api/messages` - создать новое сообщение
   - Добавить CORS для работы с фронтендом
   - Использовать yagalog для логирования
   - Настроить Dockerfile

3. **Фронтенд (React + TypeScript)**
   - Инициализировать React проект с TypeScript
   - Настроить Tailwind CSS

     - ChatApp (главный компонент)
     - MessageList (список сообщений)

4. **Docker и docker-compose**

5. **Тестирование**
   - Проверить отправку сообщений

6. **Деплой**
   - Задеплоить бекенд

7. **Саморевью**

## Технические решения

### Бекенд:
- **Фреймворк**: Стандартный `net/http` (KISS принцип) или `gin` для простоты
- **Хранение**: In-memory структура (slice с mutex для thread-safety)
- **Формат данных**: JSON
- **Логгер**: github.com/Chelaran/yagalog
- **CORS**: Настроить для работы с фронтендом

### Фронтенд:
- **Стилизация**: Tailwind CSS (быстро и эффективно)
- **State management**: React hooks (useState, useEffect)
- **HTTP клиент**: fetch API или axios
- **Структура компонентов**: Простая, без излишнего усложнения

### Docker:
- Multi-stage build для оптимизации размера
- Отдельные контейнеры для фронтенда и бекенда
- docker-compose для удобного запуска

## Принципы работы

- **DRY**: Избегать дублирования кода
- **KISS**: Простые и понятные решения
- **Эффективность**: Короткий, но качественный код
- **Поэтапные коммиты**: Каждый коммит - логический шаг работы
